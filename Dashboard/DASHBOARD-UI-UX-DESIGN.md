## Общая концепция

**Стиль:** Премиальный инвестиционный интерфейс в духе «luxury fintech», с золотыми акцентами и тёмной верхней панелью. Сочетание белого контента на светло-сером фоне и тёмного header/footer создаёт чёткую визуальную иерархию.

**Ключевые принципы:**
- Институциональный, доверительный тон
- Минималистичный layout без перегруженности
- Чёткая структура: Marquee → Header → Tabs → Content → Footer

## График циклов (Cycle Schedule)

### Назначение
Модальное окно со списком всех инвестиционных циклов 2026 года и датами открытия. Позволяет увидеть, какие циклы ещё доступны для участия, а какие уже закрыты.

### Точка входа
- Кнопка «График циклов» в правой плашке на главном экране Dashboard.
- Открытие/закрытие через `setIsScheduleOpen(true/false)`.

### Структура модального окна

| Область | Содержимое |
|---------|-------------|
| **Шапка** | Заголовок «График циклов», подзаголовок «Investment Cycles 2026», кнопка закрытия |
| **Список циклов** | Прокручиваемая область (max-h 60vh) с 14 строками |
| **Подвал** | Подсказка о дедлайне 24 часа до даты цикла |

### Логика статусов
- **Доступен (Available):** `сейчас < дата_цикла − 24 часа` — до дедлайна ещё есть время.
- **Завершен (Closed):** cutoff уже пройден — участие в цикле невозможно.

### Визуальные состояния
- **Доступен:** янтарный бейдж номера, зелёный статус с пульсирующей точкой.
- **Завершен:** серый бейдж, серый текст, бейдж «Завершен».

### Подсказка в подвале
«Средства должны быть внесены минимум за 24 часа до даты открытия цикла для участия в распределении доходности 6.8%.»

## Раскрываемый блок «Инвестиционный контракт»

### Назначение
Карточка контракта с возможностью раскрыть детали. По умолчанию показывается только заголовок и статус; по клику отображается полная информация.

### Состояния
| Состояние | Видимость | Содержимое |
|-----------|-----------|------------|
| **Свёрнуто** | Только шапка | «Инвестиционный контракт», бейдж статуса (Активен / Ожидает активации) |
| **Развёрнуто** | Полный блок | Все поля + кнопка «Скачать контракт» |

### Раскрываемые поля (список поставок / деталей)
- **Номер контракта** (`contract.number`)
- **Сумма инвестиции** (`principal`, formatCurrency)
- **Дата активации** (`activationDate` / `contract.startDate`)
- **Дата окончания** (`maturityDate` / `contract.endDate`)
- **Кнопка «Скачать контракт»** — загрузка txt-файла с текстом соглашения

### Механизм раскрытия
- **Переключатель:** кнопка «Подробнее» внизу карточки с иконкой chevron-down.
- **Анимация:** `grid-rows-[0fr] → grid-rows-[1fr]`, `opacity-0 → opacity-100`, `duration-500`.
- **Иконка:** поворот chevron на 180° при раскрытии.

### Стили
- Блок контракта: `bg-white`, `rounded-[2rem]`, `border-gray-100`.
- Зона кнопки раскрытия: `bg-gray-50/20`, `border-t border-gray-50`.
- Кнопка «Подробнее»: текст `text-amber-600`, круглый индикатор с иконкой.

## История транзакций
- **Таблица (desktop):** Дата | Тип | Сумма | Статус
- **Карточки (mobile):** тип, дата, статус, сумма
- **Цветовая схема типов:** deposit (синий), withdrawal (фиолетовый), yield (зелёный)

### Профиль
- **Секции:**
  1. Информация об аккаунте: ID инвестора, Email
  2. Личные данные: ФИО, паспортные данные, документ, Telegram
  3. Криптокошелёк: адрес
  4. Безопасность: смена пароля

### KYC Flow
- **Карточка:** белая, `rounded-[2.5rem]`
- **Поля:** Фамилия, Имя, Email, Телефон, загрузка документа
- **Кнопки:** «Подтвердить», «Пройди позже»

### Contact Binding
- **Цель:** привязка Email или Telegram перед оплатой
- **Кнопки:** «Подтвердить Email», «Привязать Telegram», «Продолжить»
